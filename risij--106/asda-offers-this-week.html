Module 04: ASDA System Analysis

Last Updated: Nov 01, 2024 Protocol ID: ASD-04

The ASDA retail system utilizes a dual-stream savings matrix: Stream A consists of instant price reductions (Rollbacks), and Stream B consists of delayed value returns via a digital wallet (ASDA Rewards). Optimal performance requires interfacing with both streams simultaneously. This guide covers the protocol for exploiting the ASDA offers this week.

Stream A: Rollback Acquisition

This is a passive data-gathering process. While navigating the retail environment, scan for green "Rollback" data markers. These indicate assets with a temporary, direct price reduction. Prioritize the acquisition of these assets as a baseline saving measure.

Stream B: Rewards API Exploitation

The ASDA Rewards application provides access to Stream B. Value is accumulated in a user's "Cashpot" by completing two types of objectives:

  • Star Products: Designated assets that return 10% of their purchase price to the user's Cashpot.
  • Missions: Gamified procurement objectives that yield a fixed Cashpot bonus upon completion (e.g., "Acquire 5 fruit assets, receive £0.50 bonus").

The Matrix Exploit: Stream Convergence

The highest-efficiency transaction occurs when Stream A and Stream B converge on a single asset. This is an item that is simultaneously on Rollback and designated as a Star Product. In this scenario, the user receives an instant price reduction at the point-of-sale, and then also receives a 10% value return on that new, lower price. Identifying these convergence points is the primary goal of this protocol. For frozen asset acquisition, cross-reference these convergence points with the Iceland offers this week for absolute market optimization.
